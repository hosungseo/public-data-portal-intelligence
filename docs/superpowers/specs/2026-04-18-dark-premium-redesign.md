# Dark Premium Redesign — Public Data Portal Intelligence

작성일: 2026-04-18
대상 페이지: `file-to-api.html` / `index.html` (동일 사본), `file-to-api.js`
현재 라이브: https://hosungseo.github.io/public-data-portal-intelligence/

## 목적과 독자

이 리디자인은 페이지의 **1차 독자를 대외 공개 쇼케이스로 재정의**하는 데서 출발한다. 기자·연구자·개발자·일반 시민이 링크를 눌렀을 때 "이 프로젝트가 무엇인지" 빠르게 이해하고, **공유하고 싶어지는** 경험을 만드는 것이 목표다. 동시에 기존의 핵심 가치인 "10,045건 파일데이터의 API 전환 검토 큐"를 만져볼 수 있는 도구성을 훼손하지 않는다.

선택된 미학은 **다크 콘솔 / Vercel Premium** 톤이다. 순검정 배경, 네온 녹색 단일 포인트 컬러, 혼합 타이포(Pretendard + Inter + SF Mono)로 개발자·제품 쇼케이스 감각과 공공데이터라는 주제를 접목한다. 히어로는 **Statement Lead**로, 첫 화면에서 "무엇을 먼저 API로 바꿀 것인가"라는 질문을 선언문으로 던진 뒤 아래로 내려가며 구체가 드러나게 한다.

## 섹션 리듬

페이지는 5개 섹션, 각 섹션 사이 여백 100–140px.

### 01 · Hero (100vh)

**상단 바**: 좌측에 녹색 점 + "Public Data Portal Intelligence" 라벨, 우측에 Queue · Method · GitHub 링크 세 개.

**본문 2열 그리드 (1.2fr : 1fr)**:
- **좌측** 선언문 3줄 ("10,045건의 파일데이터." / "무엇을 먼저 API로" 회색 / "바꿀 것인가." 녹색) + 서브카피 2문장 + CTA 두 개(녹색 "전체 큐 보기 ↓" + 고스트 "방법론")
- **우측** 4×1 KPI 그리드 (검토 후보 / 우선 검토 / 즉시 검토 가능 / 교차수요 관찰), 우선 검토 숫자만 녹색. 하단에 우선순위 1위 프리뷰 한 줄

**하단**: `↓ SCROLL` + 연도 표기. Sticky nav는 두지 않는다(상단 바와 기본 스크롤로 충분).

### 02 · 우선 검토 12건 (에디토리얼 카드)

12장 카드 2열 그리드. 1번 카드만 기본 펼침, 녹색 보더. 나머지 11장은 `--line` 보더·접힘.

각 카드:
- 상단: `01 / 12` 모노 랭크 + 레인 뱃지(우측)
- 타이틀 19px + "제공기관 · 포맷 · 주기"
- 펼침 시 "왜 먼저?" 녹색 박스가 본문 주연
- 모노 수치 라인 (↓ 다운로드 · 신호 · 메타 · API)
- 하단 "펼쳐서 근거 보기 ▾"

이 섹션은 큐 위에 놓인 **"전시"**다. 도구가 아니라 편집된 결과.

### 03 · 전체 검토 큐 (도구)

검색창 + 제공기관/레인/포맷/정렬 드롭다운 + 초기화 버튼. 10,045건을 25건/페이지로 페이지네이션.

행 구조(아래 "큐 행" 섹션 참조).

### 04 · 큐의 모습

한 panel 안에 3블록:
- **결합 상태 분포**: 가로 막대 3개 (UMY 녹색 87.2%, UM- 흰색 9.6%, -MY 흰색 3.0%)
- **주요 포맷**: 칩 5~8개 (csv/xlsx/jpg/shp/pdf), 각 칩에 건수·퍼센트 모노
- **기관 상위 5**: 리스트 5행, 우측에 건수·퍼센트 모노, 하단 "기관 전체 보기 ↗"

기존 별도 기관 섹션은 여기로 흡수.

### 05 · 방법론

기본 접힘(`<details>`). 클릭 시 기존 "공개 요약 자산 / 이 페이지가 하지 않는 일 / 추가 검토 필요 / 공개 범위" 2×2 그리드 노출.

## 큐 행 (가장 많이 클릭되는 요소)

3상태 × 3열 그리드 `40px | 1fr | auto`:

### 접힘
- 보더 `--line` 1px
- 좌: `01` 모노 랭크, 색 `--muted-deep`
- 중: "제목" 15px 500 + "제공기관 · 포맷" 11px `--muted-deep`, 그 아래 모노 한 줄 `↓ 360,588 · 신호 394,025 · 메타 3/5 · 결합 UMY`. 값만 흰색, 라벨은 `--muted`.
- 우: 레인 뱃지 + "펼치기 +"

### 호버
- 보더 `--line-hover`
- 배경 `--surface`
- 랭크 숫자도 흰색으로

### 펼침
- 보더 `--accent`
- 헤더 영역 배경 `--surface-muted`
- 본문: 좌측 녹색 바 + 요약 14px · 이유 번호 리스트 (모노 01/02/03/04 + Inter 본문) · 녹색 박스 "왜 먼저 검토?" (수치 인용은 SF Mono 11px로 괄호 안) · 하단 녹색 링크 "↗ 공공데이터포털에서 검색"

### 레인 뱃지 2단계
- **기본**: `--line-hover` 보더, 흰 텍스트
- **강조(Metadata-ready 레인에서)**: `--accent` 보더, 은은한 녹색 배경, 녹색 텍스트

## 모션 · 인터랙션

- 호버 전환 **140ms ease** — 보더·배경·텍스트
- 펼침은 **native `<details>`** — 애니메이션 없음, 즉각 반응
- 섹션 reveal: `IntersectionObserver`로 진입 시 14px 올라오며 opacity 0→1, **320ms ease**
- 링크 호버: 밑줄 대신 `opacity 0.8 → 1.0`
- 커서 점멸 등 "진짜 터미널" 은유는 **쓰지 않는다** — 색·폰트까지만

## 반응형

| 폭 | 변화 |
|---|---|
| ≥1080px | 기본 레이아웃 |
| 640–1080px | 히어로 2열 → 1열 세로 스택, KPI 4→2, 섹션 04 2열 → 1열 |
| <640px | KPI 4→2, 선언문 26px로 축소, 큐 행의 레인 뱃지+랭크 분리(우측 아이콘) |

## 접근성

- 대비: `--ink` 대 `--bg`는 WCAG AAA. `--muted` 대 `--bg`는 AA 이상. `--muted-deep` 대 `--bg`는 라벨 전용 AA 경계
- 포커스: 모든 상호작용 요소에 `outline: 2px solid var(--accent); outline-offset: 2px`
- `aria-expanded`는 `<details>`가 내장 처리
- 폰트 크기는 rem 기반, 사용자 줌에 반응

## 구현 범위와 제외

### 포함
1. 색·타이포 토큰 교체 (기존 베이지/카페 → 다크 프리미엄)
2. 히어로 구조 재구성 (선언문 + 4 KPI + 우선순위 1위)
3. 섹션 순서 재배치 및 기관 섹션의 큐의 모습 흡수
4. 우선 검토 카드 재디자인 (에디토리얼 톤, 1번 기본 펼침)
5. 큐 행 재디자인 (랭크 모노, 모노 수치 한 줄, 3상태)
6. 큐의 모습 결합 바·칩·기관 탑5 통합
7. 방법론 접힘
8. 반응형 브레이크포인트 재설정
9. 폰트 추가(Pretendard Variable CDN)

### 제외 (차기 라운드)
- 공유용 OG 이미지 제작
- 상세 차트(제공기관 715개 시각화 등)
- 애니메이션 히어로(텍스트 스테거 등)
- 한영 토글
- 실제 공공데이터포털 딥링크 재검증

## 성공 기준

1. **첫 화면 이해**: 링크를 처음 누른 사람이 5초 안에 "이 페이지가 무엇을 보여주는가"를 문장으로 말할 수 있다
2. **공유 가능성**: 헤드라인 스크린샷이 트위터·슬랙에 붙었을 때 맥락 없이도 읽힌다
3. **도구성 보존**: 10,045건 검색·필터·확장이 기존과 동등하게 작동한다
4. **스크롤 길이**: 현재(~7,455px) 대비 20% 이내 변동
5. **모바일 정상성**: 640px 폭에서 KPI가 깨지지 않고 히어로가 한 화면에 들어간다
